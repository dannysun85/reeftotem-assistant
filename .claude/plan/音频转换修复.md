# 音频转换修复计划

## 任务目标
修复前端音频转换逻辑，解决音频样本值过小导致的4007腾讯云ASR解码失败问题

## 问题描述
- **当前问题**：前端发送的音频数据值在0-33之间，远低于正常的16位音频范围(-32768到32767)
- **根本原因**：Float32到Int16转换逻辑错误，音频电平过低
- **影响**：腾讯云返回4007"音频解码失败"错误，WebSocket连接在发送音频后关闭

## 技术栈
- 前端：Tauri 2.0 + React + Web Audio API
- 后端：Rust + WebSocket + 腾讯云ASR API
- 音频格式：16kHz, 16bit, mono PCM

## 修复方案
采用最小化修复方案，专注于音频转换逻辑的修复
