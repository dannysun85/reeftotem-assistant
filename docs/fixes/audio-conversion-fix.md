# 音频转换修复计划

## 任务目标

修复前端音频转换逻辑，解决音频样本值过小导致的 4007 腾讯云 ASR 解码失败问题。

## 问题描述

- **当前问题**: 前端发送的音频数据值在 0-33 之间，远低于正常的 16 位音频范围 (-32768 到 32767)
- **根本原因**: Float32 到 Int16 转换逻辑错误，音频电平过低
- **影响**: 腾讯云返回 4007 "音频解码失败"错误，WebSocket 连接在发送音频后关闭

## 技术栈

- 前端：Tauri 2.0 + React + Web Audio API
- 后端：Rust + WebSocket + 腾讯云 ASR API
- 音频格式：16kHz, 16bit, mono PCM

## 修复方案

采用最小化修复方案，专注于音频转换逻辑的修复。

## 关键实现

### Float32 到 Int16 转换函数

```typescript
function float32ToInt16(float32Array: Float32Array): Int16Array {
  const int16Array = new Int16Array(float32Array.length);
  for (let i = 0; i < float32Array.length; i++) {
    // 正确的转换：Float32 范围 [-1.0, 1.0] → Int16 范围 [-32768, 32767]
    const s = Math.max(-1, Math.min(1, float32Array[i]));
    int16Array[i] = s < 0 ? s * 0x8000 : s * 0x7FFF;
  }
  return int16Array;
}
```
